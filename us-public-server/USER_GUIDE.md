# 灾害监测系统 - 用户使用指南

## 📱 访问系统

### 前台公开页面
访问: `http://your-server.com/` 或 `http://your-server.com/public`

**无需登录**，即可查看：
- 全球实时灾害事件
- 事件统计数据
- 按类型、国家、严重程度筛选
- 事件详细信息

### 管理后台
访问: `http://your-server.com/admin`

**需要管理员账号登录**，功能包括：
- 系统概览仪表板
- 事件管理和处理
- 成品池查看
- 日报生成和发布
- 系统管理和配置

## 🔄 工作流程说明

### 自动化流程

```
1. 定时抓取 (每天凌晨2点)
   └─> 从RSOE-EDIS抓取新事件
   └─> 写入数据库和全局事件池
   └─> 自动去重

2. 蓄水池处理 (每小时)
   └─> pending → pool（获取坐标）
   └─> pool → 下载卫星影像（GEE）
   └─> pool → 质量评估（GPT-4o-mini）
   └─> checked → 入GPU队列

3. GPU Worker拉取任务
   └─> 下载影像
   └─> 执行7项AI推理
   └─> 回传结果
   └─> 写入成品池

4. 日报生成 (每天早上7点)
   └─> Gemini生成单事件摘要
   └─> Gemini综合生成日报
   └─> 保存至数据库
```

### 手动干预场景

**场景1：加速处理特定事件**
1. 进入"灾害事件"页面
2. 找到目标事件
3. 点击"推进"按钮
4. 系统立即处理该事件

**场景2：立即获取最新数据**
1. 进入"灾害事件"页面
2. 点击"立即抓取"
3. 等待1-2分钟
4. 刷新页面查看新数据

**场景3：紧急生成日报**
1. 进入"灾害日报"页面
2. 点击"生成昨日日报"
3. 等待3-5分钟（取决于事件数量）
4. 刷新页面查看新日报

## 📊 数据解读

### 事件状态说明

| 状态 | 说明 | 预期停留时间 |
|------|------|--------------|
| 待处理 | 刚抓取到，尚未获取详细信息 | < 1小时 |
| 蓄水池 | 已有坐标，等待/正在下载影像 | 1-24小时 |
| 质检通过 | 影像质量合格，等待入队 | < 1小时 |
| 已入队 | 等待GPU Worker拉取 | 取决于队列长度 |
| 推理中 | GPU正在分析 | 30-120秒 |
| 已完成 | 分析完成，结果已保存 | 永久 |

### 严重程度判定

- **极高**：大规模灾害，重大人员伤亡或财产损失
- **高**：显著影响，需要紧急响应
- **中**：中等影响，需要持续监测
- **低**：轻微影响，常规监测

## ⚠️ 常见问题

### Q: 为什么有些事件没有影像？
A: 可能原因：
1. 云覆盖过高，GEE无法找到合适影像
2. 该地区无卫星覆盖
3. GEE配额已用完

### Q: 事件状态卡在"蓄水池"很久？
A: 可能原因：
1. GEE下载队列繁忙
2. 该地区影像稀缺
3. 云覆盖阈值设置过严

解决方法：
- 调整 `config.json` 中的 `cloud_threshold`
- 手动点击"推进"重试

### Q: 日报生成失败？
A: 检查：
1. Gemini API配额是否充足
2. 网络连接是否正常
3. 日志文件中的错误信息

### Q: 如何修改定时任务时间？
A: 
1. 编辑 `config.json`
2. 修改 `scheduler` 部分
3. 重启服务
